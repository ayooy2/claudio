# AI 电台 + 个人网页项目完整技术概览

## 一、网站运行原理：三层模型

所有网站本质上都由三层构成：

plaintext

```
浏览器（前端） ───→ 服务器（后端） ───→ 数据库
  用户看到的        处理业务逻辑        持久化存储数据
```

- **前端**：HTML/CSS/JavaScript，负责页面渲染和用户交互
- **后端**：处理核心业务逻辑，向前端提供数据接口（API）
- **数据库**：存储用户信息、歌单、播放记录等结构化数据

> 你的 AI 电台项目需要完整三层架构；纯展示型个人网页可仅用前端实现。

## 二、技术栈选型（当前主流方案）

### 前端框架（三选一）

|        框架        |                    核心特点                     |          适用场景           |
| :----------------: | :---------------------------------------------: | :-------------------------: |
|  React + Next.js   | 生态最完善，Next.js 内置路由、SSR/SSG、部署工具 | AI 电台等交互复杂的动态项目 |
|     Vue + Nuxt     |            上手门槛低，中文社区活跃             |    偏好 Vue 语法的开发者    |
| 纯 HTML + 后端模板 |               无框架依赖，最简单                |  静态个人网页、纯展示页面   |

**建议**：AI 电台优先使用 Next.js (React)；个人网页可复用同一技术栈，也可选择纯静态方案。

### 后端方案

|          方案           |                          适用场景                           |
| :---------------------: | :---------------------------------------------------------: |
| Next.js 自带 API Routes |           小型项目，前后端同仓，无需独立部署服务            |
| Python FastAPI / Flask  | AI 逻辑密集型项目（调用模型、音频处理），Python AI 生态最优 |
|     Node.js Express     |            全栈 JavaScript 开发，前后端语言统一             |

**建议**：涉及大量 AI 调用时，采用 **Python FastAPI 后端 + Next.js 前端** 组合；简单项目可直接用 Next.js API Routes 一站式开发。

### 数据库方案

|   类型   |       代表产品        |                      适用场景                      |
| :------: | :-------------------: | :------------------------------------------------: |
|  关系型  |  PostgreSQL, SQLite   |         数据结构明确（用户、歌单、评论等）         |
|  文档型  |        MongoDB        |             数据结构多变，需要快速迭代             |
| 托管服务 | Supabase, PlanetScale | 不想自行运维数据库，需要开箱即用的 Auth / 实时功能 |

**建议**：起步阶段优先使用 **Supabase**（免费额度充足，自带认证、数据库、实时订阅）或 **SQLite**（单文件数据库，零配置）。

## 三、开发到上线完整流程

1. **本地开发**
   - 在本地编写代码，通过 `npm run dev` 启动开发服务器预览
   - 使用本地 SQLite 或远程测试数据库进行开发调试
2. **版本管理**
   - 使用 Git 进行代码版本控制
   - 每次完成独立小功能后提交 commit 并推送到 GitHub
3. **自动化部署**
   - 代码推送到 GitHub 后，部署平台自动拉取、构建并上线
   - 推荐平台：Vercel（Next.js 首选，免费）、Railway/Render（后端 + 数据库）
   - 纯静态个人网页可使用 GitHub Pages（完全免费）
4. **域名与 HTTPS**
   - 部署平台默认提供免费二级域名（如 `xxx.vercel.app`）
   - 可在 Namesilo/Cloudflare 购买自定义域名，在部署平台绑定后自动配置 HTTPS
5. **持续维护**
   - 采用 "修改代码 → 提交推送 → 自动部署" 的迭代模式
   - 逐步添加监控、日志系统，排查线上问题

## 四、AI 电台项目特有技术点

1. **音频播放**：前端使用原生 `` 标签或 Howler.js 库，重点处理播放列表切换、缓冲管理、进度控制
2. **AI 集成**：后端调用 OpenAI/Claude API 实现歌单推荐、DJ 串词生成等功能，需做好 API 限流和结果缓存以控制成本
3. **实时性**：仅在需要 "多人同步收听" 的直播场景下使用 WebSocket；普通点播功能无需
4. **音频存储**：AI 生成的音乐文件需存储在对象存储服务中（推荐 Cloudflare R2，免费额度高且无出站流量费）

## 五、常见踩坑指南（按出现概率排序）

1. **CORS 跨域问题**：前后端域名 / 端口不同时浏览器会拦截请求，需在后端配置正确的 CORS 响应头
2. **环境变量缺失**：本地 `.env` 文件中的 API 密钥等敏感信息，部署时需在平台配置面板中同步设置
3. **依赖版本冲突**：锁定 `package.json` 中的依赖版本，避免随意升级导致的兼容性问题
4. **数据库连接失败**：线上环境需检查 IP 白名单、SSL 配置和数据库访问权限
5. **构建失败**：Linux 服务器区分文件名大小写，而 Windows 不区分，需确保代码中路径大小写一致
6. **API 调用超时**：AI API 响应通常较慢（几十秒），需调大前端和部署平台的超时时间，或改用流式响应
7. **热重载失效**：Next.js Fast Refresh 偶尔卡住，可清除浏览器缓存或重启开发服务器
8. **SEO / 分享预览异常**：需正确配置 Open Graph 和 Twitter Card 元标签，确保社交媒体分享时显示正确的预览图和描述

## 六、建议开发顺序

1. 使用官方脚手架初始化项目，确保本地能正常运行并看到页面
2. 开发最小可行产品（MVP）：个人网页先搭建单页框架；AI 电台先实现本地音频播放功能
3. 接入第一个后端功能：例如 AI 电台对接 OpenAI API 实现简单的歌单推荐
4. 完成首次部署，验证公网可访问性
5. 采用小步迭代模式，每次只添加一个功能，完成后立即部署验证
