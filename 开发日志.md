## 2026-06-24

### 彻底修复媒体错误 code=4（音频代理 Range 支持）

**commit: 6f57d63**

**根因分析（4 Agent 并行诊断）：**

音频代理端点 `/api/audio-proxy` 声明 `Accept-Ranges: bytes` 但不处理客户端的 Range 请求。
当浏览器播放音频时，会发送 `Range: bytes=0-` 请求，代理返回 200 OK（完整文件）而非
206 Partial Content，浏览器丢弃响应 → 资源加载失败 → code=4。

**修复内容：**

| 层级 | 修复项 |
|------|--------|
| 音频代理 | Range 请求透传到上游，返回 206 + Content-Range |
| 音频代理 | 缓冲第一个 chunk 做 magic byte 验证后再写响应头 |
| 音频代理 | 添加 WAV (RIFF) magic byte 检测 |
| 音频代理 | Content-Type 拦截 text/html/text/plain 错误响应 |
| 音频代理 | 错误处理检查 headersSent 避免 "headers already sent" |
| 前端 play() | 快速路径移到 audio.src 赋值后，用 audio.currentSrc 比较 |
| 前端 play() | code=4 时自动调用 /api/song-url?force=true 重新获取 URL |
| 后端 API | getSongUrl 支持 force 参数绕过缓存 |
| 前端 onended | 添加 .catch() 防止 unhandled rejection |
